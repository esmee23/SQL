Actualizar | Update

La sentencia "UPDATE" se utiliza para modificar los valores de una tabla.

La sintaxis en SQL de UPDATE es:

UPDATE nombre_tabla
SET campo1 = valor1, campo2 = valor2
WHERE campo = valor3

Dónde:
UPDATE: Indica la acción a realizar.
SET: Establece los nuevos valores para las columnas indicadas.
WHERE: Es la condición que tiene por objetivo seleccionar las filas que queremos modificar.

Aclaración: Si omite la cláusula WHERE, se actualizarán todos los registros de la tabla.

Ejemplo #1:

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 21
3 Antonio Hernández 22

Sí queremos cambiar la EDAD y el APELLIDO de Jacob de la tabla USUARIOS, la sentencia sería la siguiente:

UPDATE USUARIOS
SET Edad = 23, Apellido = 'Gómez'
WHERE id = 2

Ahora la tabla USUARIOS quedaría de la siguiente manera:

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Gómez 23
3 Antonio Hernández 22
Ejemplo #2:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
3 Firulais 3 Café Macho

Sí queremos cambiar el id de el perrito con NOMBRE Firulais de la tabla PERRITOS, la sentencia sería la siguiente:

UPDATE PERRITOS
SET id = 4
WHERE Nombre='Firulais' and Edad=3

Ahora la tabla PERRITOS quedaría de la siguiente manera:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho

Dónde | Where

La cláusula "WHERE" se puede usar para determinar qué registros de la tabla se quieren seleccionar.

La sintaxis en SQL de WHERE es:

WHERE Campo=[Condición] AND Campo=[Condición] OR Campo=[Condición]

Dónde:
WHERE: Es la condición que tiene por objetivo seleccionar las filas que queremos modificar, seguido de las condiciones que queremos que nuestra consulta ejecute.
AND: El operador AND mostrará los resultados cuando se cumplan las 2 condiciones.
OR: El operador OR mostrará los resultados cuando se cumpla alguna de las 2 condiciones.

Otras condiciones:
> Mayor.
>= Mayor o igual.
< Menor.
<= Menor o igual.
>= Mayor o igual.
= Igual.
<> o != Distinto.
IS [NOT] NULL Para comprobar si el valor de una columna es o no es nula, es decir, si contiene o no contiene algún valor.

Aclaración: Si no se emplea esta cláusula, la consulta devolverá todas las filas de la tabla. Al utilizar WHERE este debe ir a continuación de FROM.

Ejemplo "=":

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22
3 Antonio Hernández 22

Sí queremos mostrar todos los usuarios dónde su EDAD es 22 de la tabla USUARIOS, la sentencia sería la siguiente:

SELECT *
FROM USUARIOS
WHERE EDAD = 22

Ahora la consulta mostraría lo siguiente:

id Nombre Apellido Edad
2 Jacob Pérez 22
3 Antonio Hernández 22
Ejemplo "AND":

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22
3 Antonio Hernández 22

Sí queremos mostrar todos los datos del usuario dónde su EDAD es 22 y su APELLIDO es Hernández de la tabla USUARIOS, la sentencia sería la siguiente:

SELECT *
FROM USUARIOS
WHERE EDAD = 22 AND APELLIDO='Hernández'

Ahora la consulta mostraría lo siguiente:

id Nombre Apellido Edad
3 Antonio Hernández 22

Ejemplo "OR":

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22
3 Antonio Hernández 22

Sí queremos mostrar todos los Usuarios que tengan NOMBRE Mark o su APELLIDO sea Peréz de la tabla USUARIOS, la sentencia sería la siguiente:

SELECT *
FROM USUARIOS
WHERE NOMBRE='Mark' OR APELLIDO='Peréz'

Ahora la consulta mostraría lo siguiente:

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22

Eliminar | Delete

La sentencia "DELETE" sirve para borrar filas de una tabla.

La sintaxis en SQL de DELETE es:

DELETE FROM
WHERE EDAD = 20

Dónde:
DELETE: El comando DELETE sirve para deshacernos de los registros de una tabla.
FROM: Esta especifica los orígenes de los datos.

Aclaración: Si omite la cláusula WHERE, se eliminarán todos los registros de la tabla.

Ejemplo #1:

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22
3 Antonio Hernández 22

Sí queremos elimnar todos los datos de los usuarios que tengan NOMBRE Antonio, la sentencia sería la siguiente:

DELETE
FROM USUARIOS
WHERE NOMBRE='Antonio'

Ahora la consultamos los datos de la tabla con la siguiente instrucción:

SELECT *
FROM USUARIOS

Esta consulta mostraría lo siguiente:

id Nombre Apellido Edad
1 Mark Lopéz 20
2 Jacob Pérez 22

Insertar | Insert

La sentencia "INSERT" se utiliza para insertar nuevas filas en una tabla.

La sintaxis en SQL de INSERT es:

INSERT INTO nombre_tabla (columna1, columna2, columna3,.)
VALUES (valor1, valor2, valor3,)

Dónde:
INSERT : Indica la acción a realizar.
INTO : Se utiliza para indicar que los datos se insertaran dentro de una tabla.
VALUES : Es la condición que tiene por objetivo seleccionar las filas que queremos modificar.

Aclaración: Si omite la cláusula WHERE, se actualizarán todos los registros de la tabla.

Ejemplo #1:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho

Sí queremos insertar una nueva mascota a la tabla PERRITOS, la sentencia sería la siguiente:

INSERT INTO PERRITOS(Nombre, Edad, Color, Género)
VALUES ('Niebla', 8,'Gris', 'Hembra')

Ahora la tabla USUARIOS quedaría de la siguiente manera:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho
5 Niebla 8 Gris Hembra
Ejemplo #2:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho
5 Niebla 8 Gris Hembra

Sí queremos insertar dos mascotas mas a la tabla PERRITOS, la sentencia sería la siguiente:

INSERT INTO PERRITOS(Nombre, Edad, Color, Género)

VALUES ('Wilson', 4,'Café', 'Macho'), ('Destiny', 5,'Blanco', 'Hembra');

Ahora la tabla USUARIOS quedaría de la siguiente manera:

id Nombre Edad Color Género

2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho
5 Niebla 8 Gris Hembra
6 Wilson 4 Café Macho
7 Destiny 5 Blanco Hembra

Ordenar | Order

ORDER BY se utiliza para ordenar los resultados de una consulta, según el valor de la columna especificada.

La sintaxis en SQL de ORDER es:

SELECT nombre_columna(s)
FROM nombre_tabla
ORDER BY nombre_columna(s) ASC|DESC

Dónde:
ORDER : Indica la acción a realizar.
BY : Indica el campo que usará para ordenar el valor de la columna.
ASC|DESC : Especifica el formato del orden (Ascendente o descendente)

Aclaración: Si omite la especificar si el orden será ascendente o descendente, por defecto se ordenarán de forma ascendente según los valores de la columna.

Ejemplo #2:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
2 Pongo 7 Negro Macho
3 Danna 1 Blanco Hembra
4 Firulais 3 Café Macho
5 Niebla 8 Gris Hembra
6 Wilson 4 Café Macho
7 Destiny 5 Blanco Hembra

Sí queremos ordenar los datos por EDAD de manera ascendente la tabla PERRITOS, la sentencia sería la siguiente:

SELECT *
FROM PERRITOS
ORDER BY Edad ASC

Ahora la tabla PERRITOS quedaría de la siguiente manera:

id Nombre Edad Color Género
3 Danna 1 Blanco Hembra
1 Kira 2 Café con blanco Hembra
4 Firulais 3 Café Macho
6 Wilson 4 Café Macho
7 Destiny 5 Blanco Hembra
2 Pongo 7 Negro Macho
5 Niebla 8 Gris Hembra

Búsqueda de datos

LIKE: Es un operador lógico utilizado para determinar si una cadena específica coincide con un patrón específico. Por lo general, se usa en la instrucción "Where" para encontrar un patrón específico de una columna.

La sintaxis en SQL de LIKE es:

SELECT *
FROM nombre_tabla
WHERE nombre_columna(s) LIKE '%A%'

Con la combinación de estos caracteres podremos obtener múltiples patrones de búsqueda. Por ejemplo:

El nombre empieza por A: Nombre LIKE 'A%'
El nombre acaba por A: Nombre LIKE '%A'
El nombre contiene la letra A: Nombre LIKE '%A%'
El nombre empieza por A y después contiene un solo carácter cualquiera: Nombre LIKE 'A_'
El nombre empieza una A, después cualquier carácter, luego una E y al final cualquier cadena de caracteres: Nombre LIKE 'A_E%'

Ejemplo::

Sí queremos mostrar todos los datos de las mascotas donde su nombre termina con A de la tabla PERRITOS, la sentencia sería la siguiente:

SELECT *
FROM PERRITOS
WHERE Nombre LIKE '%A'

Ahora la consulta mostraría lo siguiente:

id Nombre Edad Color Género
1 Kira 2 Café con blanco Hembra
3 Danna 1 Blanco Hembra
5 Niebla 8 Gris Hembra

BETWEEN : El operador BETWEEN selecciona valores dentro de un rango determinado. Los valores pueden ser números, texto o fechas.

La sintaxis en SQL de BETWEEN es:

SELECT *
FROM nombre_tabla
WHERE nombre_columna(s) BETWEEN value1 AND value2

Ejemplo::

Sí queremos mostrar todos los datos de las mascotas donde su edad sea entre 3 a 8 años de la tabla PERRITOS, la sentencia sería la siguiente:

SELECT *
FROM PERRITOS
WHERE Edad BETWEEN 3 AND 8

Ahora la consulta mostraría lo siguiente:

id Nombre Edad Color Género
2 Pongo 7 Negro Macho
4 Firulais 3 Café Macho
5 Niebla 8 Gris Hembra
6 Wilson 4 Café Macho
7 Destiny 5 Blanco Hembra
